# Scope

## Purpose

Define shared naming authority for OpenClaw-managed projects.

## In Scope

- formal words
- field names
- status values
- schema keys
- file-name tokens
- forbidden synonyms
- review rules for introducing or changing formal names

## Out of Scope

- project-specific implementation logic
- repository-specific architecture decisions
- storage retention policy
- unrelated documentation style debates

## Owner Intent

Keep naming explicit, stable, reusable, and centrally reviewable.
